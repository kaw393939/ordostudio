# Studio Ordo — Recommended Sprint Sequencing

This is the shortest path from current capabilities to a working managed marketplace.

## Sprint A — Apprentice profiles + directory
- Profiles (public) + admin approval
- Referral code shown on profile

## Sprint B — Deal model + maestro assignment/approval
- Intake → Deal conversion
- Deal statuses + audit events
- Admin/maestro queue UI

## Sprint C — Offers + pricing governance (SKU)
- Offer catalog in DB
- Admin offer editor (controlled)

## Sprint D — Stripe upfront checkout + payment → deal state
- Checkout sessions
- Webhook processing
- Refund tooling

## Sprint E — Ledger + commission + payout approvals
- Ledger rows
- Admin approvals
- Exports

## Sprint F — Newsletter subscribers + scheduled sends
- Public subscribe/unsubscribe
- Send runs + delivery logging

## Sprint G — Connect payouts + contractor onboarding
- Connect onboarding states
- Execute payouts
