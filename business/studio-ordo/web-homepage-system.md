# Studio Ordo — Homepage System (Swiss / Corporate)

This is the recommended homepage module system for Studio Ordo.

## Module order

### Hero
- Headline: Bring order to AI in software delivery.
- Subhead: Spec-driven training + evaluation gates for reliable AI-augmented engineering.
- Proof strip: 20+ years teaching engineers · 10,000+ students
- CTAs:
  - Primary: View training tracks
  - Secondary: Book a technical consult

### 01 — Method (Ordo)
- 4 steps: Spec → Tests → Build → Evaluate/Audit
- Show artifacts: template preview, rubric, evaluation checklist

### 02 — Outcomes
- Leader outcomes: reliability, speed, standards
- Individual outcomes: capability, portfolio artifacts

### 03 — Offers
- Workshops
- Team program
- Advisory/enablement

### 04 — Studio (Bottega)
- Apprenticeship narrative
- Field work and critique

### 05 — Proof
- Numbers now (20+ years, 10,000+ students)
- Case studies later

### Close
- One clear next step and a low-friction fallback (email capture for templates)

## Conversion rules
- One primary CTA per viewport.
- Proof always near a decision point.
- No vague promises; show artifacts.

## Visual rules
- Strict grid, left alignment, whitespace.
- Numbered sections.
- No ornamental gradients or busy hero art.

## Implementation alignment
- Tokens: `src/app/globals.css`
- UI system: `docs/design-system.md`
- Icons: `docs/icon-map.md`
